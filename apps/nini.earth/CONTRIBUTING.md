# Contributing

Thanks for your interest in contributing!

## How to contribute
1. Fork the repo and create a feature branch: `git checkout -b feature/my-feature`
2. Make your changes following the coding standards in `docs/development.md`
3. Update or add documentation (`docs/api.md`, `docs/troubleshooting.md`, and README links) as needed
4. Commit using Conventional Commits
5. Open a Pull Request

## Commit messages
Follow Conventional Commits:
```
feat(scope): add capability
fix(ui): resolve visual issue
docs(api): update reference
```

## Development
- Install: `npm install`
- Start dev server: `npm run serve`
- Build: `npm run build`

See `docs/development.md` for details.
