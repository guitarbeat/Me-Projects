# Bookmark Organization Standards

> **Core principle:** Organize by *actionability*, not by topic.  
> "Where will I *look* for this?" beats "What is this *about*?"

---

## The Three Zones

Your three Floccus-synced folders map to **actionability levels**:

| Folder | Zone | Purpose | Review Cadence |
|--------|------|---------|----------------|
| **Bookmarks bar** | 🔥 Active | Current work. Changes weekly. | Daily |
| **Other bookmarks** | 📚 Reference | Organized knowledge. Stable. | Monthly |
| **Mobile bookmarks** | 📥 Capture | Quick saves. Inbox. | Weekly |

### Why This Works
- **One question to sort:** "Am I actively using this?" → Yes = Bookmarks bar, No = Other bookmarks
- **Mobile = Inbox:** Capture fast, process later (reduces decision friction on mobile)
- **Bookmarks bar stays small:** Only current projects = fast access

## Folder Rules

### Naming Convention
```
📁 Level 1: 🎨 Word        (emoji + one word, max 10 folders)
📁 Level 2: 📝 Word        (emoji + one word)
📁 Level 3: Word           (no emoji, optional)
```

**Examples:**
- ✅ `🎨 Creative` → `🎵 Music` → `Ableton`
- ❌ `🎨 Creative Projects & Hobbies` → `🎵 Music Production Tools`

### Limits (Johnny Decimal Inspired)
| Rule | Limit | Why |
|------|-------|-----|
| Level 1 folders | **Max 10** per zone | Scannable at a glance |
| Total depth | **Max 3 levels** | Deeper = lost bookmarks |
| Folder name | **1 word** (Level 1-2) | Forces clarity |

## Bookmark Title Standard

### Core Rule
Titles should be **short, specific, and skimmable**. Prefer what the page *is* over where it came from.

### Format
- **Target length:** 3-8 words (soft max 60 characters)
- **Preferred style:** Title Case
- **Allowed punctuation:** `:` and `&` when useful
- **Avoid:** trailing platform suffixes like `- Google Docs`, `| Reddit`, `- Wikipedia`, `- GitHub`

### Keep vs Remove
| Keep | Remove |
|------|--------|
| Main subject/topic | Site name suffixes |
| Key method/model name | Marketing taglines |
| Specific artifact type (Figure, Protocol, Dashboard) | UTM/query noise from URLs |
| Version/year when meaningful | Captcha/system text (e.g., "Just a moment", "Access Denied") |

### Rename Patterns
| Noisy Pattern | Standardized Title Pattern |
|---------------|----------------------------|
| `X - Google Docs` | `X` |
| `X | SiteName` | `X` |
| slug-style-title-with-hyphens | `Readable Title` |
| Very long natural-language query bookmark | concise intent label (e.g., `Alb-mSc KI Mice & AAV Availability Search`) |

### Examples
| Before | After |
|--------|-------|
| `HazeOver for Mac - GitHub Student Developer Pack` | `HazeOver for Mac` |
| `Fiber Amplifier Blemish Diagnosis - Google Docs` | `Fiber Amplifier Blemish Diagnosis` |
| `LMArena | Benchmark & Compare the Best AI Models` | `LMArena` |
| `Nature2023ImaginingFutureOpticalMicroscopyEverythingEverywhereAllAtOnce` | `Imagining the Future of Optical Microscopy` |

### Maintenance
- Apply title cleanup at capture time when possible.
- During weekly inbox processing, rename newly saved noisy titles before filing.
- During monthly review, batch-fix remaining slug-like or overlong titles.

## Zone 1: Bookmarks Bar (🔥 Active)

**What goes here:** Links you're using *this week*.

| Folder | Contains | Example |
|--------|----------|---------|
| 🔥 Current | Today's work | Active project dashboards |
| 📝 Writing | Active manuscripts | Thesis draft, paper in review |
| 🔬 Research | Active experiments | Current protocols, open data |

**Pruning rule:** If untouched for 2 weeks → Archive or move to Other bookmarks.

---

## Zone 2: Other Bookmarks (📚 Reference)

**What goes here:** Organized reference material you'll return to.

### Category Structure (Max 10)

| # | Folder | Contains | NOT here |
|---|--------|----------|----------|
| 1 | 💻 Dev | Code, tools, APIs, GitHub | Research code (→ Bookmarks bar) |
| 2 | 🎨 Creative | Music, art, design, media | Active creative projects (→ Bookmarks bar) |
| 3 | 🎓 Learning | Courses, tutorials, how-tos | Academic papers (→ Bookmarks bar) |
| 4 | 🏡 Life | Banking, legal, profiles | — |
| 5 | 🏢 Work | Career, professional | Active work projects (→ Bookmarks bar) |
| 6 | 🧠 Wellness | Mental health, fitness | — |
| 7 | 📰 Reading | Articles, blogs, news | Papers to cite (→ Bookmarks bar) |
| 8 | 🛒 Shopping | Wishlists, deals | — |
| 9 | 🖨️ Maker | 3D printing, hardware, DIY | — |
| 10 | 📦 Archive | Completed, inactive | — |

### The Archive Folder
Everything has a lifecycle. When you're done with something:
1. Move from Bookmarks bar → Other bookmarks (if useful reference)
2. Move to 📦 Archive (if no longer needed but want to keep)
3. Delete (if truly useless)

## Zone 3: Mobile Bookmarks (📥 Capture)

**What goes here:** Quick saves. This is your **inbox**.

| Folder | Purpose |
|--------|---------|
| 📥 Inbox | Default capture point. Process weekly. |
| ⭐ Pinned | Frequently accessed on mobile |

**Processing rule:** Every Sunday, empty the inbox:
- Move to correct folder in Other bookmarks
- Or delete if no longer relevant

---

## Decision Tree

When saving a bookmark, ask **one question**:

```
Am I actively working on this?
    │
    ├── YES → Bookmarks bar (which project?)
    │
    └── NO → Will I need this again?
              │
              ├── YES → Other bookmarks (which category?)
              │
              └── MAYBE → Mobile bookmarks > 📥 Inbox
```

**If you hesitate more than 3 seconds** → 📥 Inbox. Process later.

## Maintenance Rituals

| Cadence | Task | Time |
|---------|------|------|
| **Daily** | Scan Bookmarks bar. Still active? | 1 min |
| **Weekly** | Process Mobile inbox | 5 min |
| **Monthly** | Review Other bookmarks. Archive stale folders. | 15 min |
| **Quarterly** | Full audit. Delete unused. Merge duplicates. | 30 min |

---

## Anti-Patterns (What NOT to Do)

| ❌ Don't | ✅ Do Instead | Why |
|----------|---------------|-----|
| Create folder for every topic | Max 10 categories | Paralysis of choice |
| Nest 4+ levels deep | Max 3 levels | Lost bookmarks |
| Multi-word folder names | One word + emoji | Forces clarity |
| Keep everything "just in case" | Archive or delete | Cruft slows search |
| Save without processing | Use inbox, process weekly | Decision debt |
| Duplicate across zones | One bookmark, one home | Sync conflicts |

| **Limit depth** | Max 3 levels |
